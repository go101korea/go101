Go 소개

Go는 구글에서 만든 컴파일된 정적 타입 프로그래밍 언어 입니다. Go 디자인 및 도구 개발 팀의 많은 핵심 팀원들은 프로그래밍 언어 연구 분야에서 수년간의 경험을 가지고 있습니다.

Go는 많은 기능이 있습니다. 일부는 고유하게 작성된 것이고 일부는 다른 언어에서 차용하였습니다
  • 내장 동시 프로그래밍 지원
    • 쉽게 사용할 수 있는 그린 스레드인 고루틴
    • 채널(CSP 모델 기반) 및 고루틴 간 동기화를 위한 select 메커니즘
  • 컨테이너 타입의 map 과 slice 는 일급 객체입니다.
  • 인터페이스를 통한 다형성
  • 인터페이스를 통한 박싱 및 리플렉션
  • 포인터
  • 함수 클로저
  • 메서드
  • 디퍼드 함수 호출
  • 타입 임베딩
  • 타입 추론
  • 메모리 안정성
  • 자동 쓰레기 수집기(garbage collection)
  • 크로스 플랫폼 호환성
  • 사용자 정의 제네릭(Go 1.18 이상)
위의 기능 외에도 추가 하이라이트는 다음과 같습니다
  • Go는 다른 프로그래밍 언어와 구문이 유사하고, 단순하고 깔끔하도록 설계되었습니다. 이를 통해 Go 프로그래밍을 쉽게 익힐 수 있습니다.
  • Go는 모든 종류의 공통 기능을 제공하는 훌륭한 표준 코드 패키지 세트와 함께 제공됩니다. 대부분의 패키지는 크로스 플랫폼입니다.
  • Go에는 활발한 커뮤니티도 있고 많은 고품질의 타사 Go 패키지 및 프로젝트를 가져와서 사용할 수 있습니다.

Go 프로그래머는 고퍼들(Gophers)이라고 불립니다.

실제로 Go는 컴파일되고 정적 타입의 프로그래밍 언어이지만 Go에서는 일반적으로 동적 스크립트 언어에서만 사용할 수 있는 많은 기능도 있습니다. 두 종류를 하나의 언어로 합치는 것은 어렵지만 Go에서는 해냈습니다. 즉, Go는 정적 언어의 엄격함과 동적 언어의 유연성을 모두 가지고 있습니다. 둘 사이에 문제가 없었다고 말할 수는 없지만 Go에서의 이점 보다 훨씬 덜합니다.

Go의 디자인에서 가독성은 크게 영향을 미치는 중요한 요소입니다. 고퍼가 다른 고퍼가 작성한 Go 코드를 이해하는 것은 어렵지 않습니다.

현재 가장 인기 있는 Go 컴파일러는 Go로 작성되었으며 Go 디자인 팀에서 유지 관리합니다. 이후로 이를 표준 Go 컴파일러 또는 gc라고 부릅니다. (가비지 컬렉션 GC가 아닌 Go 컴파일러의 약자). Go 디자인 팀은 두 번째 Go 컴파일러인 gccgo도 유지 관리합니다. 요즘에는 gc보다 사용 빈도가 낮지만 항상 참조용으로 사용됩니다. 두 컴파일러 모두 활발하게 개발 중입니다. 현재 Go 팀은 gc의 개선에 집중하고 있습니다.

gc는 Go Toolchain에서 제공됩니다. (Go 팀에서 관리하는 Go 개발용 도구 세트). Go Toolchain 1.0은 2012년 3월에 출시되었습니다. Go 버전은 Go Toolchain 버전과 일치합니다. 매년 두 개의 주요 버전이 릴리즈됩니다.

Go 1.0 출시 이후 Go의 구문은 거의 바뀌지 않았지만 Go Toolchain의 도구는 버전마다 변경이 있었고, 특히 gc에 대해 많은 개선이 있었습니다. 예를 들어 쓰레기 수집으로 인한 지연은 자동 메모리 관리가 있는 언어에 대한 일반적인 문제입니다. 그러나 Go 1.8 이후로gc의 동시 쓰레기 수집에 대한 개선 사항은 기본적으로 지연 문제를 해결했습니다.

gc는 크로스 플랫폼 컴파일을 지원합니다. 예를 들어 Linux OS에서 Windows 실행 파일을 빌드할 수 있으며 그 반대도 가능합니다.

Go 언어로 작성된 프로그램은 대부분 매우 빠르게 컴파일됩니다. 컴파일 시간은 개발의 즐거움을 위한 중요한 요소입니다. 짧은 빌드 시간은 많은 프로그래머가 Go로 프로그래밍하는 것을 좋아하는 이유 중 하나입니다.

Go 실행 파일의 장점은 다음과 같습니다
  • 작은 메모리 사용량
  • 빠른 코드 실행
  • 짧은 준비 시간(훌륭한 배포 경험)
C/C++/Rust와 같은 일부 다른 컴파일된 언어도 이러한 세 가지 이점을 가질 수 있습니다. (그리고 그들은 Go에 비해 각각의 장점이 있을 수 있습니다), 그러나 그들은 Go의 세 가지 중요한 특성이 부족합니다.
  • 빠른 컴파일로 만족스러운 로컬 개발 경험 및 짧은 배포 반복 주기 제공
  • 동적언어와 같은 유연성
  • 내장 동시 프로그래밍 지원

위의 모든 이점이 결합되어 Go를 뛰어난 언어로 만듭니다. 많은 종류의 프로젝트에 적합한 선택입니다. 현재 Go는 네트워크, 시스템 도구, 데이터베이스 개발 및 블록체인 개발 분야. Go 1.18에 사용자 정의 제네릭이 도입되면서 일부 다른 영역에서 Go가 점점 더 많이 사용될 것으로 예상됩니다. GUI/게임, 빅데이터, AI 등.

마지막으로 Go는 모든 측면에서 완벽하지는 않습니다. Go 디자인에는 확실히 트레이드오프가 있습니다. 그리고 Go에는 실제로 몇 가지 단점이 있습니다. 예를 들어 Go는 이제 임의의 불변 값을 지원하지 않으므로 표준 패키지에서 수정할 의도가 없는 많은 값이 변수로 선언됩니다. 이 문제는 일부 Go 프로그램에서 잠재적인 보안 약점이 될 수 있습니다.
